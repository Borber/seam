### SBtream-Server：

1. 作为一个Spring Boot 项目来进行 只对外提供 RestFul  接口

2. 使用 `post`  `/{pid}/{id}` 来接受请求  返回同意的请求体

   1. 前期 扫描插件目录 以及 `Bull` 进行 相应的 `json#1` 生成 

   2. `pid` 作为 作为 在调用服务时 在 `json#1` 中查找对应类，进行请求 

   3. 返回为标准返回体 

      ```json
      {
          
      }
      ```

        


   

3. 插件化 使用 jar 作为插件格式 定义统一的 接口 使用 `Bull` 接口 定义统一结构

4. 可以本地运行 也可以在服务器上暴露出来

5. 可以后续加入订阅系统 对应的用户 可以有不同的订阅
